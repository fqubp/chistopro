# Деплой на Timeweb (виртуальный хостинг)

## 1) Подготовка

1. Скопируй `.env.example` в `.env`.
2. Заполни переменные:
   - `APP_ENV=production`
   - `DB_*` под параметры базы на хостинге
   - `ADMIN_PASSWORD_HASH` (сгенерируй локально)
   - `NOTIFY_EMAIL`

Генерация хэша:

```bash
php -r "echo password_hash('СЛОЖНЫЙ_ПАРОЛЬ', PASSWORD_DEFAULT) . PHP_EOL;"
```

## 2) Создание проекта на Timeweb

1. Создай сайт в панели Timeweb.
2. Подключи домен (например, `chistopro.ru`) к этому сайту.
3. Выпусти SSL (Let's Encrypt) в панели.

## 3) База данных

1. Создай MySQL-базу и пользователя.
2. Импортируй `database/schema.sql` через phpMyAdmin.
3. Проверь доступ, указав правильные `DB_*` в `.env`.

## 4) Загрузка файлов

1. Загрузи проект в корень сайта (`public_html` или каталог сайта в Timeweb).
2. Убедись, что `.env` загружен, но не доступен извне.
3. Права на `uploads/`: запись для веб-сервера (обычно 755/775, в зависимости от окружения).

## 5) Финальная проверка

1. Открой главную страницу и отправь тестовую заявку.
2. Проверь, что запись появилась в админке.
3. Проверь смену статусов/удаление заявок.
4. Проверь загрузку файла и что файл открывается по ссылке.

## 6) Базовый security checklist

- [ ] `APP_ENV=production`
- [ ] установлен `ADMIN_PASSWORD_HASH`
- [ ] включён SSL
- [ ] `.env` не в репозитории
- [ ] `test_db.php` на production закрыт (403)
- [ ] выполнение PHP в `uploads/` запрещено
